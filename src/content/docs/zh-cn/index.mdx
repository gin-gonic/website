## 什么是Gin?

Gin是一个使用Go语言开发的Web框架。 它提供类似Martini的API，但性能更佳，速度提升高达40倍。 如果你是性能和高效的追求者, 你会爱上 Gin。

<CardGrid>
	<Card title="快速" icon="rocket">
		基于 Radix 树的路由，小内存占用。没有反射。可预测的 API 性能。
	</Card>
	<Card icon="seti:plan" title="支持中间件">
		传入的 HTTP 请求可以由一系列中间件和最终操作来处理。例如：Logger，Authorization，GZIP，最终操作 DB。
	</Card>
	<Card icon="seti:lock" title="Crash 处理">
		Gin 可以 catch 一个发生在 HTTP 请求中的 panic 并 recover 它。这样，你的服务器将始终可用。例如，你可以向 Sentry 报告这个 panic！
	</Card>
	<Card icon="approve-check-circle" title="JSON 验证">
		Gin 可以解析并验证请求的 JSON，例如检查所需值的存在。
	</Card>
	<Card icon="puzzle" title="路由组">
		Gin帮助您更好地组织您的路由，例如，按照需要授权和不需要授权和不同API版本进行分组。此外，路由分组可以无限嵌套而不降低性能。
	</Card>
	<Card iconicon="open-book" title="错误管理">
		Gin 提供了一种方便的方法来收集 HTTP 请求期间发生的所有错误。最终，中间件可以将它们写入日志文件，数据库并通过网络发送。
	</Card>
	<Card icon="seti:json" title="内置渲染">
		Gin 为 JSON，XML 和 HTML 渲染提供了易于使用的 API。
	</Card>
	<Card icon="seti:html" title="可扩展性">
		新建一个中间件非常简单，去查看[示例代码](https://gin-gonic.com/zh-cn/docs/examples/using-middleware/)吧。
	</Card>
</CardGrid>
