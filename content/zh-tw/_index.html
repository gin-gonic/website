{{< /blocks/cover >}} {{% blocks/lead color="white" %}} **什麼是Gin?** Gin是一個使用Go語言開發的Web框架。 它提供類似Martini的API，但性能更佳，速度提升高達40倍。 如果您追求性能和良好的生產力，您一定會愛上Gin。 {{% /blocks/lead %}} {{< blocks/section color="light" >}} {{% blocks/feature icon="fa fa-tachometer-alt" title="Fast" %}} 基於Redix樹的路由器, 內存佔用小。不使用反射。 API性能可預測。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-bars" title="Middleware support" %}} 傳入的HTTP請求可以被一系列中間件和最終操作處理。 例如：Logger, 授權，GZIP，最終向數據庫發送信息。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-life-ring" title="Crash-free" %}} Gin可以捕獲HTTP請求過程中的panic並恢復它。這樣一來您的服務器將保持可用。 Gin也可以報告這個panic，例如，將其發送到Sentry。 {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="white" >}} {{% blocks/feature icon="fa fa-check-circle" title="JSON validation" %}} Gin可以解析和驗證請求中的JSON，例如，檢查必須字段是否存在。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-users-cog" title="Routes grouping" %}} Gin幫助您更好地組織您的路由，例如，按照需要授權和不需要授權和不同API版本進行分組。此外，路由分組可以無限嵌套而不降低性能。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-briefcase" title="Error management" %}} Gin幫助您方便地收集HTTP請求期間發生的所有錯誤。最終，中間件可以將它們寫入日誌文件、數據庫並通過網絡發送它們。 {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="info" >}} {{% blocks/feature icon="fa fa-images" title="Rendering built-in" %}} Gin為JSON、XML和HTML呈現都提供了易於使用的API。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-code" title="Extendable" %}} 創建一個新的中間件非常簡單，詳情參照示例代碼。 {{% /blocks/feature %}} {{< /blocks/section >}}
