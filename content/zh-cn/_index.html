{{< /blocks/cover >}} {{% blocks/lead color="white" %}} **什么是Gin?** Gin是一个使用Go语言开发的Web框架。 它提供类似Martini的API，但性能更佳，速度提升高达40倍。 如果您追求性能和良好的生产力，您一定会爱上Gin。 {{% /blocks/lead %}} {{< blocks/section color="light" >}} {{% blocks/feature icon="fa fa-tachometer-alt" title="Fast" %}} 基于Redix树的路由器, 内存占用小。不使用反射。 API性能可预测。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-bars" title="Middleware support" %}} 传入的HTTP请求可以被一系列中间件和最终操作处理。 例如：Logger, 授权，GZIP，最终向数据库发送信息。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-life-ring" title="Crash-free" %}} Gin可以捕获 HTTP 请求过程中的 panic 并恢复它。这样一来您的服务器将保持可用。 Gin 也可以报告这个 panic，例如，将其发送到 Sentry。 {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="white" >}} {{% blocks/feature icon="fa fa-check-circle" title="JSON validation" %}} Gin可以解析和验证请求中的JSON，例如，检查必须字段是否存在。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-users-cog" title="Routes grouping" %}} Gin帮助您更好地组织您的路由，例如，按照需要授权和不需要授权和不同API版本进行分组。此外，路由分组可以无限嵌套而不降低性能。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-briefcase" title="Error management" %}} Gin帮助您方便地收集HTTP请求期间发生的所有错误。最终，中间件可以将它们写入日志文件、数据库并通过网络发送它们。 {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="info" >}} {{% blocks/feature icon="fa fa-images" title="Rendering built-in" %}} Gin为JSON、XML和HTML呈现都提供了易于使用的API。 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-code" title="Extendable" %}} 创建一个新的中间件非常简单，详情参照示例代码。 {{% /blocks/feature %}} {{< /blocks/section >}}
